# 时间大师需求文档

## 1. 项目定位

`TimeMaster-Widget` 是一个面向 macOS 桌面的悬浮倒计时挂件，目标是提供接近原生桌面小组件风格的视觉体验，同时保留可手动设置目标日期、透明度和界面语言的能力。

当前项目以个人使用为主，但代码结构已经整理为适合持续迭代和公开发布的形式。

## 2. 当前已实现能力

### 2.1 核心功能

- 支持设置一个自定义目标日期
- 目标日期按天展示，不再要求用户设置具体小时
- 展示四条信息：
  - 目标剩余
  - 今日剩余
  - 本月剩余
  - 本年剩余
- 每一行都带对应的进度条
- 支持中文 / 英文切换
- 支持透明度调整
- 支持本地配置持久化
- 支持专注（Focus）倒计时：在设置中按分钟或小时填写时长，保存后立即开始；正常结束后播放短暂烟花动画，动画结束后挂件恢复为普通倒计时视图
- 专注完成后累计到本地统计：按自然日记录专注总时长与完成次数
- 右键菜单可打开独立「统计」窗口，查看近 30 天每日专注时长与次数及合计

### 2.2 交互行为

- 挂件为无边框、置顶、可拖动窗口
- 左键拖动位置
- 双击打开设置
- 右键打开菜单
- 右键菜单支持：
  - 打开设置
  - 打开专注统计窗口
  - 切换语言
  - 退出程序

### 2.3 视觉表现

- 当前定稿尺寸为 `173 x 173`（相对早期版本整体缩小约 10%）
- 卡片使用圆角、轻阴影、低饱和暖灰粉色系
- 右上角使用单猫装饰图
- 中文版为当前视觉基准版本
- 英文版使用独立布局参数，避免与中文版共用同一套边距导致错位

## 3. 当前已确认的设计约束

- 中文版当前效果视为基准，不应轻易被后续改动破坏
- 英文版允许单独微调布局
- 目标日期保持按天设置，不恢复小时模式
- 字号不应随意缩小
- 进度条和文字保持“整块居中、内部左对齐”的设计
- 素材图需要保持干净，不出现明显白边、黑边或残留横线

## 4. 当前代码层面的稳定结论

- 项目已经从单文件重构为多模块结构，适合继续维护
- 本地运行配置使用 `time_master_config.py`
- 仓库中只保留 `time_master_config.example.py` 作为示例
- 依赖改为标准 Python 依赖管理，不再以提交 `.pyside6_vendor/` 为发布方案

## 5. 后续常见迭代方向

以下内容适合继续作为后续需求：

- 继续微调中文 / 英文布局
- 增加新的配色主题
- 增加更多尺寸规格
- 增加启动脚本、应用打包或 `.app` 发布
- 将配置迁移到更标准的本地应用目录
- 增加截图、演示图和更完整的公开仓库说明

## 6. 变更时建议遵循的原则

- 先确认改动是针对中文版、英文版，还是两者都改
- 视觉微调优先改布局常量，不要先动功能逻辑
- 不要把本地运行状态文件提交到 Git（含 `time_master_focus_stats.json` 等个人统计）
- 新增功能前先更新这份需求文档，记录目标和边界
